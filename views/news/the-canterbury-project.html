Canterbury ディストリビューションが生誕したことを謹んで発表します。Canterbury は以前は Arch Linux, Debian, Gentoo, Grml, openSUSE と呼ばれていたコミュニティディストリビューションが合併して生まれました。プロプライエタリなオペレーティングシステムに対抗するために勢力を結集し、フリーソフトウェアコミュニティが協力して共通の目標を目指すことを知らしめます。

Canterbury は技術的には Arch のようにシンプルで、Debian のように安定しており、Gentoo のように柔軟性を備え、Grml のように強固なライブフレームワークを搭載し、openSUSE のように公平なものになります。

Canterbury Project への参加について Arch Linux の開発者 Pierre Schmitz は次のように意見を表明しています: "Arch Linux は常にテクノロジが限りなくシンプルであるように尽力してきました。ひとつのディストリビューションに集合することにより、開発者やユーザー、そして上流プロジェクトから複雑な作業が劇的に減るでしょう。Canterbury は Linux ディストリビューションの革命的な次の一歩です。"

Gerfried Fuchs (去年の openSUSE カンファレンスで Debian に関するトークを行った人物) は次のように述べました: "DEX (Debian Derivatives Exchange) は原理的に良いアイデアでしたが、その視野は狭すぎます。私たちは成功のためにさらに先に進む必要があるのです。"

Gentoo インフラチームのリード開発者である Robin H. Johnson による SCALE9x の Gentoo コア開発者のパネルでの発言: "私はコンパイルによってダウンタイムが生まれるのが大嫌いです。ちょっとキーを押すだけでパッケージをインストールできる方法を探し求めてきました。成功を収めているディストリビューションの成果を活用することで、システムメンテナンスの苦役から開放されるのです。"

Michael Prokop (Grml live CD 創設者) は Canterbury について次のように発言しています: "私たちは grml-live による普遍的なライブビルドフレームワークの作成に成功しました。フリーソフトウェアの拡大を促進する、どこでも使えるディストリビューションを心待ちにしています。"

去年の openSUSE カンファレンスでは "Collaboration Across Borders" というトピックで話し合いが持たれました。尊敬を集めているコミュニティのメンバー Klaas Freitag は次のように話しています: "カンファレンスのモットーが国際的に決められたことは、カンファレンスの前向きな成果だと考えています。"

Debian のプロジェクトリーダー Stefano Zacchiroli による Canterbury ディストリビューションについてのコメント: "去年、Debian は他のディストリビューションと様々な事柄について協力して作業を行いました。新たな取り組みの一部は Debian Derivatives Distribution (例: Derivatives Front Desk, DEX など) に向けられていましたが、openSUSE や Fedora など他のディストリビューションとカンファレンスおよびパネルに参加できたことを嬉しく思います。共同作業による最近の成果には自信を持っており、今をもって準備が整いました。Canterbury により、これまでの取り組みはさらに次のステップ、次世代のコミュニティディストリビューションへの統合に進みます。Canterbury に長寿と繁栄を。"

このアナウンスはあくまで始まりにすぎません。近日中に様々な変革が起こります。#cbproject ハッシュタグで twitter や identi.ca からフィードバックを送ってください。
