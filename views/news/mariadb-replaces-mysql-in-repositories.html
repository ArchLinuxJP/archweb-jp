Arch Linux の公式の MySQL のデフォルト実装は MariaDB になりました。MariaDB は MySQL とほとんど互換性を保っているため、アップグレードは特に気にせず行うことができます。ただし、互換性の懸念が全くないわけではないため、自動置換は行いません。

全てのユーザーはアップグレードすることを推奨します。近日中に MySQL は公式リポジトリから AUR に移動します。

MariaDB に切り替えたいユーザーは mariadb, libmariadbclient または mariadb-clients をインストールしてから mysql_upgrade を実行してシステムを移行してください。

移行例:

# systemctl stop mysqld
# pacman -S mariadb libmariadbclient mariadb-clients
# systemctl start mysqld
# mysql_upgrade -p

もうひとつの MySQL フォークである percona-server は [community] から利用できます。Percona Server は Oracle MySQL Enterprise により近いデータベースになりますが、MariaDB に存在する新機能が含まれていません。

[extra] の mysql 5.5.30-7 から、MySQL に依存していたパッケージは全て MariaDB を使うように再ビルドされます。他のパッケージのメンテナは MariaDB のパッケージに依存するように移行してください。

詳しい情報はメーリングリストを見てください。
