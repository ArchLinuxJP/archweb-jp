すぐに MariaDB の新しいメジャーリリースが [extra] に移行します。提供される機能を MySQL とはっきり区別するためにバージョンに変更が加わりました。今後は、公式リポジトリにある MySQL 実装を簡単に変更することはできなくなります。

MariaDB 10.0 では大きな変更が入ったため、アップグレードする前にテーブルをダンプして、アップグレード後にダンプファイルをリロードすることが (必須ではありませんが) 推奨されています。新しいバージョンにアップグレードした後は、忘れずに mysqld.service を再起動してから mysql_upgrade を実行しデータベースにエラーが出てないか確認して下さい。

また、何度もビルドが失敗するため TokuDB ストレージエンジンは無効化されました。ご不便をおかけして申し訳ありません。TokuDB は MariaDB 10.0.12-2 からまた利用できるようになりました。

アップグレードの手順と変更について詳しい情報は、MariaDB Knowledge Base や MySQL Reference Manual を参照してください。Akonadi を使っている場合、フォーラムに詳しいハウツーが載っています。
